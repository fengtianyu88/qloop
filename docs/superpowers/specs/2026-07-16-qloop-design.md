## 一、系统概述

qloop 用于管理团队中每个成员对外释放代码的全流程。核心能力包括：用户与权限管理、交付释放流程管理、大模型自动评审、首页搜索筛选、审计日志与通知。

### 部署环境

- 公司内网部署，全栈自托管，数据不出内网
- LLM 对接已有内网大模型 API

## 二、数据模型

### 三层结构：项目 → 版本 → 释放

#### 项目（Project）— 顶层实体

- 项目名称、描述
- 项目经理（系统用户，由超管/管理员指定）
- 项目组成员列表及项目级角色分配（PM 设定）
- 访问范围控制：哪些开发者/访客可访问此项目

#### 版本（Version）— 隶属于项目

- 版本号（如 v2.0）、版本说明
- PM 创建版本需求时设定：开发人员、测试人员、外部技术专家
- 外部接收方列表（自动带出邮箱，含有效期 + 访问范围配置）

#### 释放（Release）— 隶属于版本，核心流转单元

- 代码包（zip 压缩包，内含多种格式）
- 变更点说明（开发者手写的功能变更点）
- 测试报告（Word/Excel）
- 评审报告（Word/Excel，外部技术专家上传）
- 状态字段：草稿 → 代码待评 → 测试待评 → 专家待评 → 待确认 → 已释放 / 评审失败
- 3 次 LLM 评审结果（结构化 JSON：总评分、分项评分、通过/不通过结论、改进建议、风险点）
- 下载链接（有效期 + 访问范围，基于 MinIO 临时链接）

#### 代码包格式

zip 包自动解压后按文件类型差异化处理：

| 格式 | 类型 | LLM 处理方式 |
|------|------|-------------|
| C 代码（.c/.h，MATLAB 生成） | 文本 | 摘要（函数/结构体）+ 变更文件全文 |
| Python 代码（.py） | 文本 | 摘要（函数/类定义）+ 变更文件全文 |
| MATLAB m 代码（.m） | 文本 | 摘要（函数/脚本）+ 变更文件全文 |
| Simulink 模型（.slx/.mdl） | 二进制/XML | 提取模型结构（模块列表、信号流、参数） |
| MAT 文件（.mat） | 二进制 | 提取变量名、维度、数据类型等元信息 |
| PTH 模型权重（.pth） | 二进制 | 提取模型结构摘要（层名、参数量、维度） |

常见代码包组合：单一主格式 + 可选 .mat 文件
- C 代码 + .mat
- Python 代码（单独）
- .m 代码 + .mat
- Simulink 模型 + .mat
- .pth 权重（单独）

## 三、矩阵组织模型

### 双维度交叉

#### 维度一：过程域（树状层级）

部门 → 科室 → 小组

- 每个用户属于一个过程域位置（树上的单一节点）
- 管理员的管理范围基于此维度
- 超级管理员为每个管理员配置"管理范围"——指定其能管理的一个或多个组织单元（含下属层级）

#### 维度二：项目（扁平结构）

- PM 被超管/管理员指定后，可建立项目组
- 项目组是项目维度上的组织单元
- 一个用户可同时参与多个项目组
- PM 可能是某个过程域的领导，也可能不是

### 管理边界

- **管理员**：管人（过程域范围内用户的增删和系统级角色分配）+ 看项目（过程域范围内用户参与的项目，只读视角）
- **项目经理**：管项目（建立项目组，分配项目内角色）+ 管流程（项目内释放流程的创建和确认）

## 五、释放流程（7 步 + 3 次 LLM 评审）

### 流程步骤

| 步骤 | 角色 | 操作 | LLM 评审 |
|------|------|------|---------|
| 1 | 项目经理 | 创建版本需求，指定开发人员/测试人员/外部技术专家/外部接收方 | — |
| 2 | 开发人员 | 上传 zip 代码包 + 手写变更点说明 | — |
| 3 | 系统（LLM 评审①） | 自动评审代码包 + 变更点 | 通过→步骤4 / 不通过→退回开发人员 |
| 4 | 测试人员 | 上传测试报告（Word/Excel） | — |
| 5 | 系统（LLM 评审②） | 自动评审测试报告 + 代码包 | 通过→步骤6 / 不通过→退回测试人员 |
| 6 | 外部技术专家 | 上传评审报告（Word/Excel） | — |
| 7 | 系统（LLM 评审③） | 自动评审专家评审报告 | 通过→步骤8 / 不通过→退回专家 |
| 8 | 项目经理 | 确认释放（查看全部 LLM 结论 + 材料） | — |
| 9 | 系统 | 生成下载链接（含有效期 + 访问范围）+ 发邮件通知外部接收方 | — |

### 退回机制

LLM 评审不通过时：
- 自动退回对应人员（开发人员/测试人员/外部技术专家）
- 标记"评审失败"状态
- 保留失败记录（永久，供追溯审计）
- 修改后重新提交

### 随时重新评审

开发者、管理员、超级管理员可随时对指定释放或项目重新触发 LLM 评审。

## 六、LLM 自动评审方案

### 评审输出（C 方案：评分 + 建议 + 可配置规则）

**输出结构（结构化 JSON）：**
- 总评分（0-100）
- 分项评分（代码质量、变更合理性、测试覆盖等）
- 通过/不通过结论
- 详细改进建议
- 分项评估明细
- 风险点提示

**通过条件**：各分项评分均达到阈值 → 总体结论"通过"
**不通过条件**：任一分项低于阈值 → 总体结论"不通过" → 自动退回

**超管可配置：**
- 各分项通过阈值（如代码质量 ≥ 80 分）
- 评审模板和 Prompt（代码/测试/专家报告分别配置）
- 评审维度和权重（可增减评估项）

### 三次评审的输入材料

| 评审节点 | 输入材料 | 格式 |
|---------|---------|------|
| LLM 评审①（代码评审） | 代码包 + 变更点说明 | C/Python/MATLAB/Simulink/.mat/.pth + 文本 |
| LLM 评审②（测试报告评审） | 测试报告 | Word/Excel |
| LLM 评审③（专家报告评审） | 外部技术专家评审报告 | Word/Excel |

### 代码包处理流水线

1. zip 包自动解压
2. 文件类型识别
3. 差异化处理：
   - 文本类（.c/.h/.py/.m）：静态分析提取摘要（函数/结构体/类定义）+ 变更文件全文
   - 模型类（.slx/.mdl）：解析模型结构（模块列表、信号流、参数）
   - 数据类（.mat）：提取变量名、维度、数据类型等元信息
   - 权重类（.pth）：提取模型结构摘要（层名、参数量、维度）
4. 汇总结构化摘要 + 变更文件全文 + 变更点说明
5. 控制在上下文窗口内，送入 LLM 评审

### 多模型对接 + 自动回退

- 超管配置多个内网大模型（如 OpenAI 兼容接口、通义千问、自定义 API 等）
- 按评审节点选择模型（代码评审①→模型A，测试报告评审②→模型B，专家报告评审③→模型A）
- 自动回退：主模型调用失败/超时 → 备用模型1 → 备用模型2

## 七、首页与通知

### 首页

**双视图切换：**
- 释放视图（默认）：每行一个释放条目，显示项目名/版本号/开发人员/变更点/状态/LLM 评审结论
- 项目视图：每行一个项目，点进去看版本和释放

**筛选搜索维度：**
- 开发人员
- 项目名称
- 版本号
- 变更点

### 通知机制

**站内通知 + 邮件提醒，触发时机：**
- PM 分配任务时（开发/测试/专家收到通知）
- LLM 评审退回时（对应人员收到通知）
- 轮到自己处理时（收到通知）
- 释放完成时（外部接收方收到邮件 + 下载链接）

## 八、审计日志

完整审计：所有关键操作都记录，管理员可查看。

**记录范围：**
- 用户登录
- 创建/修改/删除（项目、版本、释放、用户）
- 文件上传/下载
- LLM 评审触发（含评审结果）
- 释放确认
- 邮件发送
- 角色分配/权限变更

## 九、技术架构

### 技术栈

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| 前端 | Vue 3 + Element Plus + Vite + TypeScript | 企业级 UI 组件丰富，适合管理后台 |
| 后端 | Python FastAPI | 与 LLM/AI 生态最贴合，异步高性能，丰富的文档处理库 |
| 数据库 | PostgreSQL | 成熟稳定，JSON 字段存储 LLM 评审结构化结果，全文搜索支持首页筛选 |
| 文件存储 | MinIO（内网对象存储） | 存储代码包/Word/Excel 报告，原生支持临时下载链接（含有效期 + 访问范围） |
| 任务队列 | Celery + Redis | 异步处理 LLM 评审任务（耗时长）、代码解析、文档解析 |
| LLM | 内网已有大模型 API | 数据不出内网，多模型可配 + 自动回退 |
| 邮件 | 内网 SMTP 邮件服务 | 释放通知邮件、外部接收方邮件通知 |

### 部署架构

- 公司内网部署，全栈自托管
- 所有组件（前端、后端、数据库、MinIO、Redis、SMTP）均部署在内网
- LLM 调用走内网大模型 API

## 十、外部接收方与下载链接

### 外部接收方

- PM 创建版本需求时设定外部接收方（从系统用户中自动带出邮箱）
- 可对接收人员设置有效期和访问范围

### 下载链接

- PM 确认释放后，系统自动生成下载链接
- 基于 MinIO 临时链接，支持有效期配置
- 支持访问范围控制（指定接收方可访问）
- 邮件通知发送给外部接收方，含下载链接
